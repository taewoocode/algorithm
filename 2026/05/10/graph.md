# Graph(그래프)

- `정점(Vertex)` 과 `간선(Edge)` 으로 이루어진 자료구조
- 정점들은 서로 연결될 수 있으며, 연결 관계를 간선으로 표현한다.
- 현실에서는 지도, 지하철 노선, SNS 친구 관계, 컴퓨터 네트워크 등을 그래프로 표현할 수 있다.

---

# Vertex (정점)

- 데이터를 저장하는 하나의 점
- 노드(Node) 라고도 부른다.

예시:

```text
A   B   C
```

- 사람
- 도시
- 컴퓨터
- 지하철역

같은 객체를 표현할 수 있다.

---

# Edge (간선)

- 정점과 정점을 연결하는 선
- 두 정점 사이의 관계를 의미한다.

예시:

```text
A ----- B
```

- 친구 관계
- 길 연결
- 네트워크 연결

등을 표현할 수 있다.

---

# 방향 그래프 (Directed Graph)

- 간선에 방향이 존재하는 그래프
- 한쪽 방향으로만 이동 가능

## 단방향 간선

```text
A ---> B
```

의미:

- A 에서 B 로 이동 가능
- B 에서 A 로는 이동 불가능

---

# 무방향 그래프 (Undirected Graph)

- 간선에 방향이 없는 그래프
- 양쪽 이동 가능

## 양방향 간선

```text
A ----- B
```

의미:

- A ↔ B 서로 이동 가능

---

# Degree (차수)

- 정점에 연결된 간선의 개수를 의미한다.

---

# OutDegree

- 현재 정점에서 나가는 간선 개수

예시:

```text
A ---> B
 \
  ---> C
```

- A 의 OutDegree = 2

---

# InDegree

- 현재 정점으로 들어오는 간선 개수

예시:

```text
B ---> A
C ---> A
```

- A 의 InDegree = 2

---

# Weight (가중치)

- 간선에 부여된 비용(값)
- 이동 비용이나 거리 등을 표현할 때 사용

예시:

```text
A --3--> B
A --5--> C
```

의미:

- A → B 비용 = 3
- A → C 비용 = 5

---

# Weighted Graph (가중치 그래프)

- 간선마다 비용(가중치)이 존재하는 그래프

예시:

```text
      2
A -------- B
 \        /
  \5    /1
    \  /
      C
```

---

# Tree (트리)

- 그래프의 한 종류
- 계층적인 구조를 가지는 자료구조
- 부모와 자식 관계로 구성된다.

특징:

- 사이클(Cycle)이 없다.
- 연결 그래프이다.
- 노드가 N개이면 간선은 항상 N - 1개이다.
    - ex) 2개의 노드를 잇는 간선은 1개이다.
- 루트 노드(Root Node)가 존재한다.
- 트리는 자식노드와 부모노드로 이루어진 계층적인 구조를 가지며 무방향 그래프의 일종이자 사이클이 없는 자료구조를 의미한다.
 

---

# 트리 구조 그림

```text
                A
              /   \
            B       C
          /  \     / \
         D    E   F   G
```

---

# 트리 용어

## Root Node (루트 노드)

- 가장 최상위 노드

예시:

```text
A
```

---

## Parent Node (부모 노드)

- 다른 노드를 가리키는 상위 노드


예시:

```text
A -> B
```

- A 는 B 의 부모 노드

---

## Child Node (자식 노드)

- 부모에게 연결된 하위 노드

예시:

```text
A -> B
```

- B 는 A 의 자식 노드

---

## Leaf Node (리프 노드)

- 자식이 없는 노드

예시:

```text
D E F G
```

---

# Tree 특징 그림

```text
                A
              /   \
            B       C
          /  \       \
         D    E       F
```

- A 는 Root
- B,C 는 A 의 자식
- D,E 는 B 의 자식
- D,E,F 는 Leaf Node

---

# 사이클(Cycle)이 없는 이유

- 트리는 다시 자기 자신으로 돌아오는 경로가 존재하지 않는다.

## 사이클이 있는 경우 (트리 아님)

```text
A --- B
|     |
|     |
C ----
```

- A → B → C → A 가능
- 따라서 트리가 아니다.

---

# 그래프와 트리 차이

| 구분 | 그래프 | 트리 |
|---|---|---|
| 사이클 | 있을 수 있음 | 없음 |
| 루트 노드 | 없음 | 존재 |
| 연결 구조 | 자유로움 | 계층 구조 |
| 간선 수 | 자유 | N - 1 |
| 방향 | 가능 | 보통 부모 → 자식 |
